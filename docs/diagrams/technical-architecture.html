Pipeline Flow (pipeline/run.ts)
1

Collect SERP

collect-serp.ts
  • Google Custom Search API
  • Top 20 results per keyword
  • Per country × device
  • → serp_results table
›
2

Scrape Pages

scrape.ts
  • Firecrawl API
  • Markdown + HTML head
  • Structured data (JSON-LD)
  • → snapshots table
›
3

Classify Actors

classify.ts
  • LLM: ministral-3:14b
  • actor_name, category
  • page_type, is_lacoste
  • → serp_results updated
›
4

Gap Analysis

analyze-gap.ts
  • LLM compares Top 10 vs Lacoste
  • 5 diagnostic axes + recos
  • Retry × 3 + jsonrepair
  • → analyses table
›
5

Movement Detection

analyze-movement.ts
  • Compare vs previous run
  • Detect position changes
  • New entries / exits
  • ⏸ Disabled (needs 2+ runs)
Resume mode: RESUME_RUN_ID=uuid skips steps 1-2, auto-detects completed 3-4

LLM Processing Details

Prompt Structure (V2)

// System prompt
"Expert SEO. Analyse on-site uniquement.
 JSON obligatoire. Réponses courtes."

// User prompt includes:
=== MOT-CLÉ : polo homme ===
--- Position 1 : celio.com (url) ---
META HEAD: ...
STRUCTURED DATA: ...
CONTENU MARKDOWN: ...

--- LACOSTE (Position 4) ---
or: LACOSTE (Absent — page ref)
or: LACOSTE : aucune page trouvée

Output Format (JSON → Markdown)

// LLM returns JSON array:
[{
  "keyword": "polo homme",
  "search_intent": "transactional",
  "intent_match": "Le Top 10...",
  "content_gap": "Concurrents...",
  "structure_gap": "...",
  "meta_gap": "...",
  "schema_gap": "...",
  "recommendations": ["..."],
  "tags": ["meta_title", ...]
}]

// → stored as markdown sections
// → rendered as collapsible cards
